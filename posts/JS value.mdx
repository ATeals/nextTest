# 배열

---

자바스크립트의 배열은 다른 언어와 달리 문자열, 숫자, 객체 심지어 다른 배열이나

어떤 타입의 값이든 담을 수 있답니다.

```javascript
const a = [1, "2", [3]];

a.length; // 3
a[0]; // 1
a[2][0]; // 3
```

또한 배열의 크기는 미리 정하지 않고 선언할 수 있습니다.

배열의 인덱스는 숫자이지만, 배열 자체로도 하나의 객체여서 키/프로퍼티 문자열을 추가할 수 있습니다.

(하지만 이런 코드는 혼란과 오류를 부추기기 때문에 사용하지 않는 편이 좋겠죠?)

> 추가적으로 키/프로퍼티 문자열로 추가해도 배열의 length는 늘어나지 않습니다.

```javascript
const a = [];

a["food"] = "kimch";
a["food"]; // kimch
a.length; // 0
```

<aside>
🚨 주의!
만약 배열의 키로 넣은 값이 표준 10진수 숫자로 타입이 바뀌면,
마치 문자열 키가 아닌 숫자 키를 사용한 것과 같은 결과가 발생합니다.
정말 소름끼치죠?

</aside>

```javascript
const a = [];

a["14"] = 1;
a.length; // 15
```

# 문자열

---

우리는 이름따라 문자열을 “문자의 배열”이라고 생각하기 쉽습니다. 정말 그럴까요?

아래의 두 예시로 비교해 보겠습니다.

```javascript
const a = "foo";
const b = ["f", "o", "o"];
```

문자열과 배열은 겉모습이 닮아 있습니다.(유사배열입니다. )

둘다 `length`, `indexOf()`[ES5에만 존재], `concat()`을 가지고 있습니다.

```javascript
a.length; // 3
b.length; // 3

a.indexOf("o"); // 1
a.indexOf("o"); // 1
```

와 문자열은 “문자의 배열”이 맞네요?

그렇지 않습니다.

```javascript
a[1] = "0";
b[1] = "0";

a; // "foo"
b; // ["f", "0", "o"]
```

일단 문자열은 불변 값이고 배열은 가변값입니다.

모든 자바스크립트 엔진에서 a[1]처럼 문자열의 특정문자를 접근할 수 있는 것은 아닙니다.

(요즘에는 왠만하면 될수도 있겠네요..)

여기서 핵심적인 특징은 문자열은 불변 값이므로 문자열 메서드는 그 내용을 바로 수정하지 않고,

항상 새로운 문자열을 생성후 반환합니다.

반면, 대부분의 배열 메서드는 그 자리에서 바로 원소를 수정합니다.

```javascript
const c = a.toUpperCase();

a === c; //false

b.push("!");
b; // ["f", "0", "o", "!"]
```

그리고 대부분의 배열 메서드는 문자열에 사용할 수 없지만,

문자열에 대해 불변 배열 메서드는 사용할 수 있습니다.

# 숫자

---

자바스크립트의 숫자 타입은 number가 유일하며, 정수와 부동 소숫점 숫자를 모두 포괄합니다.

## 숫자 구문

---

자바스크립트의 숫자는 10진수로 표시합니다.

```javascript
const a = 42;
const b = 42.3;

const c = 0.42;
```

소숫점 앞 정수가 0이면 생략 가능합니다.

당연히 소숫점 이하가 0일때도 생략가능합니다.

( `const d = 42.` 이렇게 작성할 수는 있지만, 남들이 볼때 혼동가능한 코드는 적지 않는게 좋겠죠?)

대부분의 숫자는 10진수가 기본값이고 소숫점 이하 0은 제거합니다.

```javascript
const a = 42.3;
const b = 42.0;

a; // 42.3
b; // 42
```

아주 크거나 아주 작은 숫자는 지수형으로 표시하고, toExponential() 메서드의 결과값과 같습니다.

(근데 이런 수를 다룰일이 많을까요..?)

```jsx
const a = 5e10;
a; // 50000000000
a.toExponential(); // "5e+10"
```

숫자 값은 Number 객채 래퍼로 박싱할 수 있기 때문에 Number.prototype 메서드로 접근할 수 있습니다.

(뭔 소린지 몰라도 나중에 나옵니다!)

예를 들면 toFixed() 메서드는 지정한 소숫점 이하 자릿수 까지 표시합니다.

```jsx
const a = 42.59;

a.toFixed(1); // "42.6"
a.toFixed(4); // "42.5900"
```

이때 메서드는 숫자에 바로 접근 할 수 있으므로 굳이 변수에 할당하지 않아도 됩니다.

```tsx
42.toFixed(3); // SyntaxError
```

엥? 바로 접근할 수 있다면서 왜 구문오류가 뜨죠? 아시겠나요? 답은 위에 나와있답니다.

에러가 난 이유는 위에서 `42.` 이놈이 올바른 숫자의 표현이기 때문입니다.

우리 자바스크립트가 `.` 를 메서드 접근자로 사용하는게 아닌 숫자의 포함되기 때문이죠….

따라서 이렇게 작성해줘야 합니다.

```tsx
(42).toFixed(3);
(42).toFixed(3);
```

둘다 같은 표현이지만 동료가 밑에 처럼 코딩 하면, 참…. ㅎㅎ

또한 숫자는 진법표현을 할 수 있습니다.

## 작은 소수 값

---

다음은 이진 부동 소수점 숫자의 문제입니다. (이건 자바스크립트만의 문제는 아닙니다.)

```tsx
0.1 + 0.2 === 0.3; //false
```

간단하게 말하면 0.3이아니라 0.300000000000000004에 “가까운” 수 라고 합니다.
