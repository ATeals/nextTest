> ⚠️ 주의! 똥글입니다. 정확한 정보 없이 작가가 써 재껴 놓은 글이므로 현명한 독자의 판단이 요구됩니다. <br /> _혹자는 이 글이 너무 두서 없어서 차라리 휴지로 쓰는 편이 유용할거라고 말합니다…._

우리는 이런저런 이유로 백준, 프로그래머스, 릿코드 등 코딩테스트 문제를 지원해주는 페이지에서 코딩테스트를 풀어봅니다. 물론 저도 풀어 보고요. _(아직 멀었다…..)_

사실 저는 코딩테스트는 테스트케이스를 통과하기만 하면 성공이라고 생각하는 편이기 때문에, 클린코드를 작성한다던지, “이 코드가 저 코드보다 낫다.”라는 평가는 코딩 테스트에서 의미 없다고 생각합니다. 그냥 성공하면 장땡이죠.

그럼 이 글을 왜 쓰고있냐? ㅋㅋㅋㅋㅋ 본론으로 넘어가겠습니다.

제가 푼 문제를 하나 예시로 보여드리죠. 문제는 다음과 같습니다.

~~https://school.programmers.co.kr/learn/courses/30/lessons/181899~~

그리고 제 코드는 이렇습니다.

```jsx
const solution = (start, end) =>
    Array(start + 1)
        .fill(start)
        .map((a, i) => a - i)
        .filter((i) => i >= end);
```

맞습니다. 저는 한줄충입니다. _두둥…_ 🤣

저는 문제를 읽으면 일단 어떤 방식으로 풀지 생각을 하고, 그 코드를 한 줄로 줄일 방법부터 생각해봅니다. 뭔가 한줄로 풀지 못하면 실패한거 같은 생각이 들때도 있죠…. _진짜 변태다…_

코드의 가독성 따위는 무차별적으로 무시하고, 그저 간지 하나만을 위해서 코드를 작성하는 방법이죠….

그치만 코드를 한줄로 작성한다는 것은 사실 생각보다 쉽지 않은 일입니다. 특히 명령형으로 코드는 한줄로 작성하는 것이 거진 불가능 하죠.. _가능하긴 한가?_

그래서 저는 코드를 한줄로 작성한다는 것은 어느정도 추상화된 코드들을 사용하는 것이라고 생각합니다.

거창하게 말했지만, 사실 저도 처음에는 한 줄 코드가 뭔가 프로그래머에게 있어서 멋있어보이는 칭호와 같아서 시작하게 되었고, 저에게 그 이유가 전부라고 생각했지만.. “함께 자라기”라는 책을 읽고 나서 제가 한 줄 코드를 집착하는 새로운 이유를 알게 되었습니다.

📌 … 때로는 말로 길게 풀었느느 것이 가독성을 떨어뜨리기도 합니다. 뭔가 장황해 보이니까요. 우리는 산문보다 운문에서 배울 것이 많습니다……. 저는 때로 가독성을 손해보면서 까지 중복을 줄이기도 합니다. 객체 지향에서 그걸 하다 보면 흥미로운 객체들을 발견합니다. 함수형에서 그렇게 하다 보면 흥미로운 함수와 함수의 함수를 발견합니다. 이 “흥미로운 무엇”은 강력합니다. 내가 전에 모르던 것을 배우게 됩니다. 그리고 종종 이 것은 프로그래머의 울타리를 넘어서 영향력을 끼치기도 합니다.

-   함께 자라기 -

</aside>

이 글은 추상화에 대한 저의 시각을 바꾸어 주었습니다. 저에게 있어서 추상화란 그저 알고싶진 않지만, 사용해야하는 로직을 이쁘게 포장해서 내가 그 코드를 짜기 위한 공부를 줄이는 방법이라고 생각했는데, 추상화를 기준으로 새로운 인사이트를 얻거나 공부할 것이 많아진다니… 정말 흥미롭습니다.

생각해보면 저도 코딩테스트에서 다른사람의 풀이들을 보면서 한 줄 코드에서 새로운 메서드들을 알게 된 것 같습니다. 사실 유심히 보지 않는다면, **reduce**나 **startWith**같은 메서드의 존재를 쉽게 알아차리지 못하기 때문이죠.

이들을 실제 코드의 적용해보고 다른 방식을 사용할때와는 다른 장단점을 알게 되면서 코딩이 훨씬 재밌고, 좀 더 공부하는 기분이 들더군요. 또한 다른사람의 코드를 더 잘 이해하게 되었습니다. 그 사람이 어떤 관점으로 코드를 작성했는지, 그 사람이 좋아하는 방식이 뭔지를 더 생각하게 되었습니다.

_특히 이부분은 지금 보고계신 블로그를 완성하는데 큰도움이 되었습니다…_

놀라운건 한줄코드를 보면서 제가 생각보다 더 코드를 잘 작성하게 되었다는 겁니다.

_물론 그렇다고 잘한다는 건 아닙니다… 알고리즘은 아직 0~1단계를 푸는 수준이고 백준도 실버니까요…._

저에게 있어서 추상화란 흥미가 생기는 판도라의 상자인 것 같습니다. 사실 같은 로직을 실행하는 코드지만,추상화라는 상자 안에 감추어져 있기 때문에 조금 더 탐구하고 싶은 욕망이 생기는 것 같거든요.

개발자의 글쓰기나 클린코드와 같이 적절한 수준의 추상화랑 로직을 보고 명령형 코드를 작성하면서 로직을 따라가는 가독성 좋은 코드들도 좋지만, 암호화된 거 마냥 작성한 추상화 한줄 코드는 저에게 코딩 흥미를 이끌어 주면서 스스로 공부하게 되는 좋은 방법인 것 같습니다.

앞으로도 저같은 한줄충이 늘어났으면 좋겠네요…. _츄릅.._

이상… 명령형 추상형 함수형 클린코드 등등 여러 자료를 보다가 머리터져서 제 생각을 작성해본 글입니다..
